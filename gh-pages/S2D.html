Class: S2D

S2D(S2Dpoint)

The Slider 2D class allows you to initialize a handle inside a "slider-2D-X" Element. This will then be able to be queried for its values within the two-dimensional space of the slider.

setRanges()

Set ranges.

This function sets the horizontal and vertical ranges of the slider. The handle will then map its percent position to the ranges provided to give the proper return values when getValues is called.
It can take One, Two, or Six arguments.

The Suffixes passed in for any of the following options can be any string. It will be returned with the values of the handle when getValues is called.
Source:
  • S2D.js, line 331
Examples

One argument:
  This must be an entire object that defines the high, low, and suffixes for both the horizontal and the vertical ranges. 

   Ex:
     Arg_1:
       {
         h: {
           low: {Number},
           high: {Number},
           suffix: {String}
         },
         v: {
           low: {Number},
           high: {Number},
           suffix: {String}
         },
       }

     // Example call:
     S2DInstance.seRanges(Arg_1);

Two arguments:
  This must be two objects that define the horizontal and vertical ranges with suffixes. 

   Ex:
     Arg_1: // The Horizontal range.
       {
         low: {Number},
         high: {Number},
         suffix: {String}
       }
     Arg_2: // The Vertical range.
       {
         low: {Number},
         high: {Number},
         suffix: {String}
       }

     // Example call:
     S2DInstance.seRanges(Arg_1, Arg_2);

Six arguments:
  This must be the low and high values of the horizontal and vertical ranges respectively. 

   Ex:
     Arg_1:   // The Horizontal low.
       {Number}
     Arg_2:   // The Horizontal high.
       {Number}
     Arg_3: // The Horizontal suffix.
       {String}
     Arg_4:  // The Vertical low.
       {Number}
     Arg_5:  // The Vertical high.
       {Number}
     Arg_6:   // The Vertical suffix.
       {String}

     // Example call:
     S2DInstance.seRanges(Arg_1, Arg_2, Arg_3, Arg_4, Arg_5, Arg_6);
